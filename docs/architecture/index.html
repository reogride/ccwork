C. Data Flow (시퀀스)

CRUD 4개 시나리오의 사용자 액션 → 컴포넌트 → Context → API → JSON Server 흐름.

sequenceDiagram
    actor User as 사용자
    participant NL as NoteList
    participant NE as NoteEditor
    participant Ctx as NotesContext
    participant API as api/notes.ts
    participant SRV as JSON Server

    Note over User,SRV: 1. 초기 로드 (fetchNotes)
    Ctx->>API: fetchNotes()
    API->>SRV: GET /notes
    SRV-->>API: Note[]
    API-->>Ctx: notes[]
    Ctx-->>NL: notes[], loading=false

    Note over User,SRV: 2. 노트 생성 (createNote)
    User->>NE: 새 노트 버튼 → 제목/내용 입력
    User->>NE: 저장 클릭
    NE->>Ctx: createNote(title, content)
    Ctx->>API: createNote({title, content})
    API->>SRV: POST /notes (+ createdAt, updatedAt)
    SRV-->>API: Note (with id)
    API-->>Ctx: newNote
    Ctx-->>Ctx: setNotes([...prev, newNote])
    Ctx-->>NL: notes[] 갱신
    NE-->>NE: onDone() → isCreating=false

    Note over User,SRV: 3. 노트 수정 (updateNote)
    User->>NL: 노트 클릭 → selectedNoteId 변경
    NL-->>NE: selectedNoteId prop
    NE-->>NE: useEffect → title/content 동기화
    User->>NE: 내용 수정 후 저장 클릭
    NE->>Ctx: updateNote(id, {title, content})
    Ctx->>API: updateNote(id, updates)
    API->>SRV: PATCH /notes/:id (+ updatedAt)
    SRV-->>API: Note (updated)
    API-->>Ctx: updated Note
    Ctx-->>Ctx: setNotes(map replace)
    Ctx-->>NL: notes[] 갱신

    Note over User,SRV: 4. 노트 삭제 (deleteNote)
    User->>NL: 삭제 버튼 클릭 (stopPropagation)
    NL->>Ctx: deleteNote(id)
    Ctx->>API: deleteNote(id)
    API->>SRV: DELETE /notes/:id
    SRV-->>API: 200 OK
    API-->>Ctx: void
    Ctx-->>Ctx: setNotes(filter)
    Ctx-->>NL: notes[] 갱신 (항목 제거)

D. State Map (상태 전이)

앱 전체 UI 상태 전이. App 로컬 상태 + NotesContext 비동기 상태 포함.

stateDiagram-v2
    [*] --> Loading : 앱 마운트

    state Loading {
        [*] --> Fetching
        Fetching --> Done : fetchNotes() 성공
        Fetching --> Errored : fetchNotes() 실패
    }

    Loading --> Idle : 로드 완료 (notes[] 준비)
    Loading --> ErrorState : 로드 실패

    state Idle {
        [*] --> Empty
        Empty --> Creating : onNewNote() 호출
        Empty --> Selecting : onSelect(id) 호출
        Creating --> Empty : 취소 / onDone()
        Selecting --> Empty : 취소 / onDone()

        state Creating {
            [*] --> Editing
            Editing --> Saving : handleSave() — title 있음
            Saving --> Done2 : createNote() 성공
            Saving --> Editing : createNote() 실패
            Done2 --> [*]
        }

        state Selecting {
            [*] --> Editing2
            Editing2 --> Saving2 : handleSave() — title 있음
            Saving2 --> Done3 : updateNote() 성공
            Saving2 --> Editing2 : updateNote() 실패
            Done3 --> [*]
        }
    }

    Idle --> Idle : deleteNote() → notes[] filter
    Idle --> Idle : createNote() 성공 → notes[] append
    Idle --> Idle : updateNote() 성공 → notes[] map replace

    ErrorState : error 상태\n(error 메시지 표시)

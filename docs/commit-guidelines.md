# 커밋 메시지 가이드

참조: [commit-messages-guide (한국어)](https://github.com/RomuloOliveira/commit-messages-guide/blob/master/README_ko-KR.md)

## 형식

```
<type>: <제목>          ← 50자 이하
                        ← 빈 줄
<본문 첫째 줄>           ← 한 줄 72자 이하
<본문 둘째 줄>           ← 최소 2줄 필수
```

## 허용 타입

| 타입       | 용도                               |
| ---------- | ---------------------------------- |
| `feat`     | 새 기능                            |
| `fix`      | 버그 수정                          |
| `docs`     | 문서 변경                          |
| `style`    | 포맷, 세미콜론 등 (코드 변경 없음) |
| `refactor` | 기능 변경 없는 코드 개선           |
| `test`     | 테스트 추가/수정                   |
| `chore`    | 빌드, 설정, 패키지 등              |
| `build`    | 빌드 시스템, 외부 의존성           |
| `ci`       | CI 설정                            |
| `perf`     | 성능 개선                          |
| `revert`   | 이전 커밋 되돌리기                 |

## 자동 검증 규칙 (commitlint)

| 규칙            | 조건      |
| --------------- | --------- |
| 제목            | 필수      |
| 제목 길이       | 50자 이하 |
| 본문            | 필수      |
| 본문 줄 길이    | 72자 이하 |
| 본문 최소 줄 수 | 2줄 이상  |

## 작성 원칙

- **명령형 동사** 사용 — "Add feature" O, "Added feature" X
- **무엇을** 했는지 제목, **왜** 했는지 본문
- 모호한 메시지 금지 — "Fix stuff", "Update" X

## 예시

```
feat: 노트 검색 기능 추가

검색어 입력 시 제목과 본문을 동시에 필터링.
debounce 300ms 적용으로 불필요한 API 호출 방지.
```
